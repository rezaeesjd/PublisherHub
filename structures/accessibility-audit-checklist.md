# Accessibility Audit Checklist (Formal)

## Keyboard-only flow
- [ ] Can tab through archive cards, pagination, and post CTAs in logical order.
- [ ] Focus state visible for all links/buttons/details summary.
- [ ] No keyboard trap in FAQ accordion.

## Contrast
- [ ] Muted text meets contrast target against background.
- [ ] Pills/chips remain readable in default and hover/focus states.

## Screen reader pass (NVDA/VoiceOver)
- [ ] Breadcrumb announces current page.
- [ ] FAQ question/answer semantics read correctly.
- [ ] Related sections have distinct headings and are navigable.

## Regression notes
- [ ] Add date/time, tester name, and URLs audited.
