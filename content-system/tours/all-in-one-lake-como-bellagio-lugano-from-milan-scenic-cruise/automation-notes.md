# Automation Notes

## Reuse Cadence (Weekly/Daily)
- Use this same package structure for every tour: `brief.md`, `keywords.md`, `blog-post.md`, `faq.md`, `meta.json`, `internal-links.md`, `automation-notes.md`.
- Recommended baseline cadence:
  - 1 booking-intent landing-blog hybrid post/week
  - 1 informational or comparison post/week
  - 1 weekly refresh of existing high-intent pages
- Scale to near-daily only when factual review capacity remains reliable.

## Scalable AI-Assisted Workflow
1. Intake product facts from supplier/OTA pages.
2. Map facts into structured fields (duration, inclusions, meeting point, policies).
3. Generate keyword clusters by intent.
4. Draft blog + FAQ from fixed template blocks.
5. Run human factual QA before publishing.
6. Publish and internally link into the Milan day-trip cluster.

## Keep Layout and Design Consistent
- Keep section order fixed in every `blog-post.md`.
- Reuse the same soft CTA + strong CTA pattern.
- Keep “Who this is best for” and “What to know before booking” in every page for conversion consistency.
- Use consistent placeholder conventions until final URLs are available.

## Adaptation to Similar Tours
This framework can be cloned for related products such as:
- Lake Como + Bellagio only tours
- St. Moritz/Bernina routes from Milan
- Verona/Sirmione/Garda day trips

Only swap verified product facts and keyword intent while preserving template logic.

## Growth Impact Alignment
This workflow supports:
- **Lead generation:** captures long-tail destination and day-trip demand.
- **Customer acquisition:** answers objections before checkout.
- **Direct bookings:** keeps `{{WebsiteLink}}` as primary CTA.
- **Sustainable growth:** repeatable system for publishing and refreshing high-intent content clusters.
